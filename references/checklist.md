# 完整性自查与交付纪律 (Checklist)

在报告生成完成后、交付前，必须逐一检查：

## 核心模块完整性

- [ ] **横向可比公司分析**：包含竞对填充，≥2家公司
- [ ] **综合股东回报率**：已计算并展示
- [ ] **三情景假设及目标价测算**：包含悲观/基准/乐观，有汇率换算提醒
- [ ] **交叉验证**：5.1模块与5.3敏感性矩阵相同CAGR与PE组合下的目标价完全一致（精确到小数点后两位）
- [ ] **技术面分析与关键价位**：验证指标真实计算（MA/RSI/Boll）
- [ ] **未来6个月关键催化剂日历**：已列出可验证事件
- [ ] **大师三视角**：7.1/7.2/7.3均已独立展开，没有被缩略或并入表格，每段≥150字
- [ ] **死穴压测**：3.0业务本质中包含≥300字的死穴分析，正负篇幅均衡
- [ ] **仓位管理模型** 和 **论据失效清单**（≥5条）已完整

## 格式与排版验证

- [ ] **查重与排版**：检查大纲序号无重复（如出现两次 `## 3.`）
- [ ] **若使用追加模式('a')修复批次**：必须用 Python 正则脚本按 `## \d+\.` 去重和重排
- [ ] **数据真实性**：所有定量数据标注来源或计算过程，无「数据缺失」被隐瞒

## 数学校验

- [ ] 目标价计算公式展示完整
- [ ] 汇率换算正确（如港股EPS人民币→港币）
- [ ] 敏感性矩阵数值交叉验证通过
